The Apollo CLI by default starts the [TUI](/docs/tui) when run without any arguments.

```bash
apollo
```

But it also accepts commands as documented on this page. This allows you to interact with Apollo programmatically.

```bash
apollo run "Explain how closures work in JavaScript"
```

---

### tui

Start the Apollo terminal user interface.

```bash
apollo [project]
```

#### Flags

| Flag         | Short | Description                                |
| ------------ | ----- | ------------------------------------------ |
| `--continue` | `-c`  | Continue the last session                  |
| `--session`  | `-s`  | Session ID to continue                     |
| `--prompt`   |       | Prompt to use                              |
| `--model`    | `-m`  | Model to use in the form of provider/model |
| `--agent`    |       | Agent to use                               |
| `--port`     |       | Port to listen on                          |
| `--hostname` |       | Hostname to listen on                      |

## Commands

The Apollo CLI also has the following commands.

---

### agent

Manage agents for Apollo.

```bash
apollo agent [command]
```

---

### attach

Attach a terminal to an already running Apollo backend server started via `serve` or `web` commands.

```bash
apollo attach [url]
```

This allows using the TUI with a remote Apollo backend. For example:

```bash
# Start the backend server for web/mobile access
apollo web --port 4096 --hostname 0.0.0.0

# In another terminal, attach the TUI to the running backend
apollo attach http://10.20.30.40:4096
```

#### Flags

| Flag        | Short | Description                       |
| ----------- | ----- | --------------------------------- |
| `--dir`     |       | Working directory to start TUI in |
| `--session` | `-s`  | Session ID to continue            |

---

#### create

Create a new agent with custom configuration.

```bash
apollo agent create
```

This command will guide you through creating a new agent with a custom system prompt and tool configuration.

---

#### list

List all available agents.

```bash
apollo agent list
```

---

### auth

Command to manage credentials and login for providers.

```bash
apollo auth [command]
```

---

#### login

Apollo is powered by the provider list at [Models.dev](https://models.dev), so you can use `apollo auth login` to configure API keys for any provider you'd like to use. This is stored in `~/.local/share/apollo/auth.json`.

```bash
apollo auth login
```

When Apollo starts up it loads the providers from the credentials file. And if there are any keys defined in your environments or a `.env` file in your project.

---

#### list

Lists all the authenticated providers as stored in the credentials file.

```bash
apollo auth list
```

Or the short version.

```bash
apollo auth ls
```

---

#### logout

Logs you out of a provider by clearing it from the credentials file.

```bash
apollo auth logout
```

---

### github

Manage the GitHub agent for repository automation.

```bash
apollo github [command]
```

---

#### install

Install the GitHub agent in your repository.

```bash
apollo github install
```

This sets up the necessary GitHub Actions workflow and guides you through the configuration process. [Learn more](/docs/github).

---

#### run

Run the GitHub agent. This is typically used in GitHub Actions.

```bash
apollo github run
```

##### Flags

| Flag      | Description                            |
| --------- | -------------------------------------- |
| `--event` | GitHub mock event to run the agent for |
| `--token` | GitHub personal access token           |

### mcp

Manage Model Context Protocol servers.

```bash
apollo mcp [command]
```

---

#### add

Add an MCP server to your configuration.

```bash
apollo mcp add
```

This command will guide you through adding either a local or remote MCP server.

---

#### list

List all configured MCP servers and their connection status.

```bash
apollo mcp list
```

Or use the short version.

```bash
apollo mcp ls
```

---

#### auth

Authenticate with an OAuth-enabled MCP server.

```bash
apollo mcp auth [name]
```

If you don't provide a server name, you'll be prompted to select from available OAuth-capable servers.

You can also list OAuth-capable servers and their authentication status.

```bash
apollo mcp auth list
```

Or use the short version.

```bash
apollo mcp auth ls
```

---

#### logout

Remove OAuth credentials for an MCP server.

```bash
apollo mcp logout [name]
```

---

#### debug

Debug OAuth connection issues for an MCP server.

```bash
apollo mcp debug <name>
```

### models

List all available models from configured providers.

```bash
apollo models [provider]
```

This command displays all models available across your configured providers in the format `provider/model`.

This is useful for figuring out the exact model name to use in [your config](/docs/config/).

You can optionally pass a provider ID to filter models by that provider.

```bash
apollo models anthropic
```

#### Flags

| Flag        | Description                                                  |
| ----------- | ------------------------------------------------------------ |
| `--refresh` | Refresh the models cache from models.dev                     |
| `--verbose` | Use more verbose model output (includes metadata like costs) |

Use the `--refresh` flag to update the cached model list. This is useful when new models have been added to a provider and you want to see them in Apollo.

```bash
apollo models --refresh
```

### run

Run apollo in non-interactive mode by passing a prompt directly.

```bash
apollo run [message..]
```

This is useful for scripting, automation, or when you want a quick answer without launching the full TUI. For example.

```bash "apollo run"
apollo run Explain the use of context in Go
```

You can also attach to a running `apollo serve` instance to avoid MCP server cold boot times on every run:

```bash
# Start a headless server in one terminal
apollo serve

# In another terminal, run commands that attach to it
apollo run --attach http://localhost:4096 "Explain async/await in JavaScript"
```

#### Flags

| Flag         | Short | Description                                                        |
| ------------ | ----- | ------------------------------------------------------------------ |
| `--command`  |       | The command to run, use message for args                           |
| `--continue` | `-c`  | Continue the last session                                          |
| `--session`  | `-s`  | Session ID to continue                                             |
| `--share`    |       | Share the session                                                  |
| `--model`    | `-m`  | Model to use in the form of provider/model                         |
| `--agent`    |       | Agent to use                                                       |
| `--file`     | `-f`  | File(s) to attach to message                                       |
| `--format`   |       | Format: default (formatted) or json (raw JSON events)              |
| `--title`    |       | Title for the session (uses truncated prompt if no value provided) |
| `--attach`   |       | Attach to a running apollo server (e.g., http://localhost:4096)  |
| `--port`     |       | Port for the local server (defaults to random port)                |

### serve

Start a headless Apollo server for API access. Check out the [server docs](/docs/server) for the full HTTP interface.

```bash
apollo serve
```

This starts an HTTP server that provides API access to apollo functionality without the TUI interface. Set `APOLLO_SERVER_PASSWORD` to enable HTTP basic auth (username defaults to `apollo`).

#### Flags

| Flag         | Description                                |
| ------------ | ------------------------------------------ |
| `--port`     | Port to listen on                          |
| `--hostname` | Hostname to listen on                      |
| `--mdns`     | Enable mDNS discovery                      |
| `--cors`     | Additional browser origin(s) to allow CORS |

```bash
apollo session [command]
```

#### list

```bash
apollo session list
```

##### Flags

| Flag          | Short | Description                          |

| `--max-count` | `-n`  | Limit to N most recent sessions      |

### stats

Show token usage and cost statistics for your Apollo sessions.

```bash
apollo stats
```

#### Flags

| Flag        | Description                                                                 |
| ----------- | --------------------------------------------------------------------------- |
| `--days`    | Show stats for the last N days (all time)                                   |
| `--tools`   | Number of tools to show (all)                                               |
| `--models`  | Show model usage breakdown (hidden by default). Pass a number to show top N |
| `--project` | Filter by project (all projects, empty string: current project)             |

---

### export

Export session data as JSON.

```bash
apollo export [sessionID]
```

If you don't provide a session ID, you'll be prompted to select from available sessions.

---

### import

Import session data from a JSON file or Apollo share URL.

```bash
apollo import <file>
```

You can import from a local file or an Apollo share URL.

```bash
apollo import session.json
apollo import https://github.com/i-luv-pho/apollov2/s/abc123
```

---

### web

Start a headless Apollo server with a web interface.

```bash
apollo web
```

This starts an HTTP server and opens a web browser to access Apollo through a web interface. Set `APOLLO_SERVER_PASSWORD` to enable HTTP basic auth (username defaults to `apollo`).

#### Flags

| Flag         | Description                                |
| ------------ | ------------------------------------------ |
| `--port`     | Port to listen on                          |
| `--hostname` | Hostname to listen on                      |
| `--mdns`     | Enable mDNS discovery                      |
| `--cors`     | Additional browser origin(s) to allow CORS |

---

### acp

Start an ACP (Agent Client Protocol) server.

```bash
apollo acp
```

This command starts an ACP server that communicates via stdin/stdout using nd-JSON.

#### Flags

| Flag         | Description           |
| ------------ | --------------------- |
| `--cwd`      | Working directory     |
| `--port`     | Port to listen on     |
| `--hostname` | Hostname to listen on |

---

### uninstall

Uninstall Apollo and remove all related files.

```bash
apollo uninstall
```

#### Flags

| Flag            | Short | Description                                 |
| --------------- | ----- | ------------------------------------------- |
| `--keep-config` | `-c`  | Keep configuration files                    |
| `--keep-data`   | `-d`  | Keep session data and snapshots             |
| `--dry-run`     |       | Show what would be removed without removing |
| `--force`       | `-f`  | Skip confirmation prompts                   |

---

### upgrade

Updates apollo to the latest version or a specific version.

```bash
apollo upgrade [target]
```

To upgrade to the latest version.

```bash
apollo upgrade
```

To upgrade to a specific version.

```bash
apollo upgrade v0.1.48
```

#### Flags

| Flag       | Short | Description                                                       |
| ---------- | ----- | ----------------------------------------------------------------- |
| `--method` | `-m`  | The installation method that was used; curl, npm, pnpm, bun, brew |

## Environment variables

Apollo can be configured using environment variables.

| Variable                              | Type    | Description                                       |
| ------------------------------------- | ------- | ------------------------------------------------- |
| `APOLLO_AUTO_SHARE`                 | boolean | Automatically share sessions                      |
| `APOLLO_GIT_BASH_PATH`              | string  | Path to Git Bash executable on Windows            |
| `APOLLO_CONFIG`                     | string  | Path to config file                               |
| `APOLLO_CONFIG_DIR`                 | string  | Path to config directory                          |
| `APOLLO_CONFIG_CONTENT`             | string  | Inline json config content                        |
| `APOLLO_DISABLE_AUTOUPDATE`         | boolean | Disable automatic update checks                   |
| `APOLLO_DISABLE_PRUNE`              | boolean | Disable pruning of old data                       |
| `APOLLO_DISABLE_TERMINAL_TITLE`     | boolean | Disable automatic terminal title updates          |
| `APOLLO_PERMISSION`                 | string  | Inlined json permissions config                   |
| `APOLLO_DISABLE_DEFAULT_PLUGINS`    | boolean | Disable default plugins                           |
| `APOLLO_DISABLE_LSP_DOWNLOAD`       | boolean | Disable automatic LSP server downloads            |
| `APOLLO_ENABLE_EXPERIMENTAL_MODELS` | boolean | Enable experimental models                        |
| `APOLLO_DISABLE_AUTOCOMPACT`        | boolean | Disable automatic context compaction              |
| `APOLLO_DISABLE_CLAUDE_CODE`        | boolean | Disable reading from `.claude` (prompt + skills)  |
| `APOLLO_DISABLE_CLAUDE_CODE_PROMPT` | boolean | Disable reading `~/.claude/CLAUDE.md`             |
| `APOLLO_DISABLE_CLAUDE_CODE_SKILLS` | boolean | Disable loading `.claude/skills`                  |
| `APOLLO_CLIENT`                     | string  | Client identifier (defaults to `cli`)             |
| `APOLLO_ENABLE_EXA`                 | boolean | Enable Exa web search tools                       |
| `APOLLO_SERVER_PASSWORD`            | string  | Enable basic auth for `serve`/`web`               |
| `APOLLO_SERVER_USERNAME`            | string  | Override basic auth username (default `apollo`) |

---

### Experimental

| Variable                                        | Type    | Description                             |
| ----------------------------------------------- | ------- | --------------------------------------- |
| `APOLLO_EXPERIMENTAL`                         | boolean | Enable all experimental features        |
| `APOLLO_EXPERIMENTAL_ICON_DISCOVERY`          | boolean | Enable icon discovery                   |
| `APOLLO_EXPERIMENTAL_DISABLE_COPY_ON_SELECT`  | boolean | Disable copy on select in TUI           |
| `APOLLO_EXPERIMENTAL_BASH_MAX_OUTPUT_LENGTH`  | number  | Max output length for bash commands     |
| `APOLLO_EXPERIMENTAL_BASH_DEFAULT_TIMEOUT_MS` | number  | Default timeout for bash commands in ms |
| `APOLLO_EXPERIMENTAL_OUTPUT_TOKEN_MAX`        | number  | Max output tokens for LLM responses     |
| `APOLLO_EXPERIMENTAL_FILEWATCHER`             | boolean | Enable file watcher for entire dir      |
| `APOLLO_EXPERIMENTAL_OXFMT`                   | boolean | Enable oxfmt formatter                  |
| `APOLLO_EXPERIMENTAL_LSP_TOOL`                | boolean | Enable experimental LSP tool            |
